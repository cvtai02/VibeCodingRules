### Any Project rules:
- Project contain: 
	1. app: main bussiness source code that expose apis
	2. ui: admin for interact with app api, configure app settings, use accesstoken in early login.
	3. handoffs: backend handoff updated api for ui, ui handoff needed api changes.
	2. api-clients: api-clients writen in typescript. (interface + implementation). so that i can copy this to any other app to use, or i can push to npm-packages
	3. api-mpc-server: can be used for create mock data, fix dirty data, smoke test, ...
	4. index.md: index for the roots folder, explain each folder use.
	4. rules.md
	5. agents instructions

	- Use dependency injection when possible.
	- Have a Core folder that define entities, aggregates, enums & constants, policies, abstractions/contracts, ...
	- Entity Class should define constrants for that entity (by using public, private properties/methods properly)
	- Aggregates defines constraints for relationship between multi-entity. 
	
	- Infrastructure for external services: Database Config, Storage, Api Clients.... that implement abstractions/constract in Core. All infrastructure should be easy to switch by app settings or user favor settings. Ex: SqlLite/Postgresql, R2/GoogleDrive...
	
	- Usecases Layer
	- Api Layer
	- Dtos should be used by both API and Usecases. both use the same dto.

	- use native fetch.
